Copy to Data Store

This tool works with an input layer to copy it to a data store. Data is copied to the ArcGIS Data Store and can be stored in a relational data store or spatiotemporal data store.

For example, you could copy features stored in a big data file share to a relational data store and specify that you only want features in the current map extent copied. This would create a hosted feature service with only the feature that were in the map extent specified.

Choose layer to copy

The layer that will be copied. This can be a dataset with a geometry (point, polyline, polygon) or a tabular dataset.

In addition to choosing a layer from your map, you can select Browse Layers found at the bottom of the drop-down list to browse your contents for a big data file share dataset or feature layer.

Choose the Data Store to copy to

The data store that the layer will be copied to. The options are:

  • Spatiotemporal—this is the default.
  • Relational.

Result layer name

This is the name of the layer that will be created in My Content and added to the map. The default name is based on the tool name and the input layer name. If the layer already exists the tool run will fail.

Using the Save result in drop-down box, you can specify the name of a folder in My Content where the result will be saved.
